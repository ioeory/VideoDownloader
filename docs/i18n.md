# 🌐 Internationalize (i18n) Workflow

VideoDownloader features a custom i18n system designed to support both English ("en") and Chinese ("zh").

## Architecture

The system uses a **Patch-based injection** mechanism rather than standard `.po` or `.json` translation files. This ensures high performance in the GUI and minimal runtime overhead.

### 1. Translation Tooling
*   **`apply_i18n.py`**: A developer tool that parses `videodownloader/gui/app.py` and injects translation dictionaries and logic.
*   **`apply_i18n_logs.py`**: Scans the engine and core files to replace hardcoded strings with localized variables.
*   **`rebuild_i18n.py`**: A master script to refresh the entire codebase's translations.

### 2. The Multi-language Map
Translations are stored in a nested dictionary within the GUI class:
```python
self.i18n = {
    "en": { "start": "Start Download", ... },
    "zh": { "start": "开始下载", ... }
}
```

### 3. Dynamic Switching
The GUI implements a `toggle_language()` method which:
1.  Flips the `current_lang` state.
2.  Calls `update_ui_texts()` to refresh all static labels.
3.  Rebuilds dynamic options (like dropdowns) to reflect the new language.

## Adding a New Language

To add support for a third language (e.g., Japanese):
1.  Open `apply_i18n.py`.
2.  Add a `"ja"` key to the `init_injection` dictionary.
3.  Translate all keys.
4.  Run `python rebuild_i18n.py`.

## Best Practices
*   **Source in Chinese**: Most developers on this project start with Chinese strings.
*   **Format Strings**: Use `.format()` or f-strings carefully to ensure they match across all languages.
*   **Log Files**: `download.log` usually follows the language set at the start of the process.
